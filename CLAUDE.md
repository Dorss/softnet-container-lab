# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

A minimal [containerlab](https://containerlab.dev/) topology demonstrating a 2-node dual-stack (IPv4 + IPv6) point-to-point lab using a custom Docker image. The primary learning goals are containerlab topology authoring, Docker-based network nodes, and clean separation of image vs. node-specific configuration.

## Key Commands

**Build the Docker image:**
```bash
bash scripts/build-image.sh basic-lab
# or directly:
cd containerlab/basic-lab && docker build -t clab-softnet-basic:latest .
```

**Deploy the topology:**
```bash
cd containerlab/basic-lab && ./deploy.sh
```

**Inspect / verify:**
```bash
cd containerlab/basic-lab
containerlab inspect -t basic-lab.clab.yml
docker exec clab-basic-lab-node1 ip addr show
docker exec clab-basic-lab-node2 ip addr show
```

**Test connectivity:**
```bash
docker exec clab-basic-lab-node1 ping -c 3 10.0.0.2       # IPv4
docker exec clab-basic-lab-node1 ping -6 -c 3 fc00::2     # IPv6
```

**Shell access:**
```bash
docker exec -it clab-basic-lab-node1 bash
```

**Destroy topology:**
```bash
cd containerlab/basic-lab && ./destroy.sh
```

## Architecture

### Lab structure

Each lab lives in its own subdirectory under `containerlab/`. Shared deploy/destroy logic lives in `containerlab/lib/` and is sourced by thin 5-line wrappers in each lab directory. Adding a new lab means creating a new directory and a wrapper — no changes to shared code.

### Lifecycle separation

The Dockerfile uses `CMD ["sleep", "infinity"]` as PID 1 — the container stays alive indefinitely. Network configuration (IP assignment, etc.) is done **after** containerlab creates the virtual links, via the `exec` directive in the topology file. This avoids polling for interface readiness: `eth1` is guaranteed to exist when the exec command runs.

### Node-agnostic entrypoint

`bin/entrypoint.sh` contains no hostname logic or hardcoded addresses. All node-specific values (`NODE_IP`, `NODE_IP6`, `PEER_IP`, etc.) come from a config file bind-mounted at `/etc/nodes/<hostname>.cfg`. The topology file mounts the right config for each node read-only (`:ro`).

### IPv6 DAD disabled

Addresses are added with the `nodad` flag to skip Duplicate Address Detection, eliminating ~1 s startup delays (safe in a lab where addresses are pre-assigned and unique).

### Config file format (`configs/node*.cfg`)

Plain bash variables sourced by the entrypoint:
```bash
NODE_IP=10.0.0.1
NODE_PREFIX=24
NODE_IP6=fc00::1
NODE_PREFIX6=64
PEER_IP=10.0.0.2
PEER_IP6=fc00::2
```

### Runtime state

`clab-basic-lab/` is auto-generated by containerlab on deploy (Ansible inventory, TLS artifacts, topology JSON). It is gitignored and should not be edited manually.

## Adding a New Lab

1. Create `containerlab/<lab-name>/` with a `Dockerfile`, topology file, `bin/entrypoint.sh`, and `configs/`
2. Write 5-line `deploy.sh` and `destroy.sh` wrappers that set `LAB_NAME`, `IMAGE`, `TOPOLOGY`, `LAB_DIR` and source `../lib/deploy.sh` / `../lib/destroy.sh`
3. Each lab uses its own image tag (e.g. `clab-softnet-<lab-name>:latest`) to stay fully independent
